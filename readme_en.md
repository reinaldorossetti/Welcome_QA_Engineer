## üéØ What a Junior QA Needs (Market Roadmap)

Focused on what job postings really ask for, here are the key skills to stand out:

### üß† Foundations and Process (priority order)

- **1. Test Discipline:** Understand test types (functional/feature, regression, smoke, etc.) and when to apply each throughout the SDLC. The core of QA is testing, process, documentation, and business understanding.
- **2. Test Planning:** Build the test plan for a release and organize regressions. The plan also covers the test suite and test data preparation.
- **3. Documentation:** Produce well-described manual scenarios and test charters.
- **4. Bug Management:** Report defects clearly with repro steps, evidence/logs, and lifecycle states.
- **5. Agile & Tools:** Know Scrum, Kanban basics, and Jira (or similar) for task management. Figma is important to validate screens against design.

### üíª Technical and Automation (priority order)

- **1. Programming Logic:** Foundation for any automation and structured reasoning.
- **2. API Testing:** Test and automate APIs with Postman, Robot Framework, or Rest-Assured. API testing is daily QA work‚Äîat least know how to do it with Postman.
- **3. SQL Databases:** Introductory querying; many SQL courses are available on Udemy.
- **4. Version Control:** Master essential Git/GitHub commands (clone, pull, push, commit).
- **5. Web Testing:** Know frameworks like Selenium, Cypress, Playwright, or Robot. Keep at least one GitHub project to show the basics; saying you know nothing can hurt interviews.

## ü§ù Soft Skills: The Junior QA Edge

Tools matter, but seniority comes from how you think, communicate, and solve problems. Modern QA influences quality. Companies hire based on technical profiles but fire due to a lack of soft skills‚Äîthis applies to everyone in tech.

**Critical behavioral competencies:**

- **Assertive, Nonviolent Communication:** Report issues without blame; explain impact clearly to devs and Product Owners.
- **Critical and Analytical Thinking:** Question requirements, assess risks, and anticipate problems.
- **User Empathy:** Be the user‚Äôs advocate; flag usability issues even without technical errors.
- **Negotiation and Persuasion:** Defend quality with risk-based arguments when delivery needs to be delayed.
- **Adaptability (Resilience):** Requirements change; stay focused and emotionally balanced in agile environments.

### üìö Must-Read Soft Skills for IT

- **Nonviolent Communication** ‚Äì Marshall B. Rosenberg
- **Soft Skills: The Software Developer‚Äôs Life Manual** ‚Äì John Sonmez
- **How to Win Friends and Influence People** ‚Äì Dale Carnegie
- **Mindset: The New Psychology of Success** ‚Äì Carol S. Dweck
- **The Clean Coder** ‚Äì Robert C. Martin
